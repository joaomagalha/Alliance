# CLAUDE.md

Guidance for Claude Code working in this repo. Read this first — it exists so you don't have to re-explore the codebase every session.

## Project Overview

Static multi-page site for **Alliance Moinho**, a Brazilian Jiu-Jitsu academy in Cuiabá, MT. All user-facing copy is **Brazilian Portuguese**. The user (João Victor) speaks Portuguese — respond in Portuguese unless asked otherwise.

## Tech Stack

Plain HTML5 + Tailwind CSS v4 + Vanilla JavaScript. **No JS framework, no bundler, no SSR.** Tailwind v4 compiles via `@tailwindcss/cli`. External CDNs: Font Awesome 6.5.0, Google Fonts (Roboto).

## CSS Build

```bash
# one-shot
npx tailwindcss -i ./Css/style.css -o ./output.css
# watch
npx tailwindcss -i ./Css/style.css -o ./output.css --watch
```

**Never hand-edit `output.css`.** If `output.css` shows up dirty in `git status`, it was regenerated by the build — leave it alone unless rebuilding is the task.

## Design Tokens (defined in `:root` of `Css/style.css`)

Use these variables. **Don't hardcode hex values.**

**Cores:**
- `--yellow #FEBC11` (brand), `--yellowSoft #FFCC47`, `--yellowDark #997000`, `--lightYellow #FFFCF5`
- `--gray #333333`, `--grayOpaque #333333d5`, `--grayMid #565656`, `--secondaryTextGray #333333a6`, `--placeholderColor #00000026`
- `--white #FFFFFF`, `--whiteOpacity #ffffffbf`
- `--red #E83636`, `--green #72A184`, `--lightGreen #00ff1e17`, `--greenPulse #00C853`
- Linhas: `--lineGray rgba(255,255,255,0.15)`, `--darkLineGray rgba(51,51,51,0.05)`

**Gradientes:** `--grayGradient`, `--grayGradientOpacity`, `--yellowGradient`

**Shadows:** `--grayShadowLarge`, `--grayShadowSmall`

**Espaçamento (rem):** `--4 0.4` `--8 0.8` `--16 1.6` `--24 2.4` `--32 3.2` `--40 4` `--48 4.8` `--56 5.6` `--64 6.4`

**Radius:** `--smallRadius 0.8` `--mediumRadius 1.6` `--largeRadius 2.4` (rem)

**Container:** `--containerLateralPadding` (2.4 mobile → 4.8 tablet → 8 desktop), `--containerSuperiorCabecalho 1.6`, `--containerSuperiorMin 3.2`, `--containerSuperiorMax 6.4`, `--maxContent 112` (rem; grows to 130/150 on huge screens)

**Z-index scale:** `--z-behind -1`, `--z-overlay 1`, `--z-card-icon 2`, `--z-card-label 3`, `--z-menu 100`

**Typography convention:** `html { font-size: 62.5% }` → **1rem = 10px**. Sizes step up at the 1024px breakpoint (h1 2.4→3.2rem, h2 1.8→2.4rem, body 1.2→1.4rem).

## Responsive Breakpoints

- `≥768px` — tablet (2-column grids, side-by-side history section)
- `≥1024px` — desktop (3-col class cards, 4-col benefits, 3-col pricing with center-emphasis, larger type)
- `≥1180px / 1440px / 1700px / 2000px` — hero height bumps; `--maxContent` grows at 1700/2000

## JavaScript — what `Js/script.js` actually does (5 things, not 3)

1. **Preloader** — fades out splash screen after `load` with a min 1600 ms display; locks body scroll while visible.
2. **Flashlight hover effect** — sets `--mouse-x` / `--mouse-y` CSS vars on `.pricingCard, .classCard, .benefitCard, .testimonialCard, .professor-card, .differentialCard` for the radial-gradient glow.
3. **On-scroll reveal** — IntersectionObserver adds `.animate` to `.animate-on-scroll` and `.animate-on-scroll-list` (one-shot, `rootMargin: "0px 0px -10% 0px"`).
4. **FAQ accordion** — only one `<details class="faqSectionCard">` open at a time.
5. **Schedule tab switcher** — `data-tab` on `.tab` targets `#<id>` tables; ARIA tablist with arrow-key navigation.
6. **Mobile menu** — open/close hamburger, outside-click/scroll/Escape close, focus management, `inert` on hidden menu. **Guarded by `if (openBtn && closeBtn && mobileMenu)` because the menu only exists in `index.html`.**

Target DOM via `data-*` attributes, not class names, for new JS behavior.

## Conventions

- Tailwind utilities for layout/spacing; CSS variables in `style.css` own brand tokens and the type scale.
- Semantic HTML (`<header>`, `<main>`, `<footer>`, `<section>`, `<details>/<summary>`) + ARIA.
- Respect `prefers-reduced-motion` — animations, hover lifts, and the flashlight are already gated on it. Maintain this when adding motion.
- Focus-visible style is global (`:focus-visible` outline yellow); don't override per-component unless there's a real reason.

## Gotchas / Things That Have Bitten Us

- **Mobile menu code is index-only** — the JS guard prevents `TypeError` on internal pages. Don't remove it.
- **`output.css` churn** — the auto-save hook commits it after most edits. Big diff there ≠ a bug.
- **Hero height ladders by breakpoint** (38→50→60→70→82→95→110 rem). Don't set a fixed height when adding hero content.
- **Pricing cards reorder on desktop** — `nth-child(2)` is rendered first (left), `nth-child(1)` center-emphasized. Markup order ≠ visual order.
- **`html` is 62.5%** — when you see `1.2rem` in CSS, that's 12px. Don't "fix" it.

## GitHub & Auto-push

Repo: **https://github.com/joaomagalha/alliance-website** (`gh` at `~/bin/gh`)

A `PostToolUse` hook in `.claude/settings.local.json` runs `git add -A && git commit -m "chore: auto-save <data hora>" && git push origin main` automatically after each file edit Claude Code makes. **Don't manually run git commands unless the user asks** — the hook handles it, and parallel manual commits cause conflicts.

If `git status` looks unexpectedly dirty, it's almost always `output.css` regenerated by Tailwind — not a stale edit.
